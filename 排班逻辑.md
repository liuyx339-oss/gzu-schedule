# 医院员工排班系统逻辑文档

## 一、系统概述

本系统是一个医院放射科和超声科（B超）的智能排班解决方案，主要功能包括：
1. 基于 Prophet 模型的需求预测
2. 智能排班生成（静态版和预测版）
3. JSON 数据导出供大屏展示

---

### 2.1 核心排班脚本

| 文件名 | 功能描述 |

## 五、排班算法核心逻辑

### 5.1 需求计算

```python
def get_hourly_headcount(workload_mins, dept, role, is_daytime):
    max_machines = 8 if dept == '放射' else 7
    hc = math.ceil(min(workload_mins, max_machines * 60) / 60.0)
    if is_daytime:
        return max(hc, 2) if role == '放射技师' else max(hc, 1)
    return 1
```

**计算逻辑**：
1. 根据预测的技师/医生操作耗时，计算每小时需要的理论人数
2. 考虑科室设备数量上限（放射8台，超声7台）
3. 白天时段：放射技师至少2人，医生至少1人
4. 夜间时段：至少1人值班

### 5.2 排班优先级规则

#### 夜班分配优先级
1. 放射医生：兼职医生 → 全职医生
2. 放射技师：全职技师
3. 超声医生：全职医生

#### 白班/半天班分配
按科室分别处理：超声医生 → 放射技师 → 放射医生

### 5.3 约束条件

#### 人员约束
- **夜班后不接白班**：前一天值夜班（N/L/N/兼职夜）的人员，次日不能排白班或半天班
- **全职医生夜班限制**：Dustin Huang 和 Li Zhenhuan 每周最多值2次夜班
- **李主任固定出诊**：周一、周四下午（H3专家门诊）

#### 排班公平性
- 使用**工时比竞争模型**：选择当前工时/目标工时比值最小的人员优先排班
- 确保工作量均衡分配

### 5.4 排班流程

```
1. 读取需求预测数据
   ↓
2. 构建需求矩阵（日期 × 24小时 × 科室角色）
   ↓
3. 预排李主任出诊（周一、周四 H3）
   ↓
4. 遍历每个日期：
   ├─ 分配夜班（先扣减夜间需求）
   ├─ 分配白班（迭代覆盖白天需求）
   └─ 记录人员工时
   ↓
5. 生成排班矩阵
   ↓
6. 转换为 JSON 供大屏展示
```

## 七、B超科专属排班（generate_bchao_roster.py）

### 7.1 规则
- 每天根据业务类型需求不同人数：
  - 周一/三/五/六：体检需求 **2人**
  - 周二/四/日：妇产科需求 **3人**
- 按工时公平排序，优先分配工时最少的人员
- 班次分配：
  - 2人时：白班 D（8.5h）
  - 3人时：部分人排晚白班 D2（8h）

### 7.2 李主任固定出诊
- 周一、周四下午：H3（13:30-17:30）专家门诊

## 九、模型特点

1. **双版本并行**：静态版保留原始班表，预测版使用优化班表
2. **需求驱动**：基于历史数据和 Prophet 预测生成排班
3. **公平分配**：工时比竞争模型确保工作量均衡
4. **灵活约束**：支持人员偏好和业务规则配置
5. **大屏兼容**：JSON 输出直接对接可视化大屏
